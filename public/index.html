Userdata folder

Your settings, presets, accounts, and Steam auth tokens live in a userdata folder. Pointing this somewhere stable means your data survives version upgrades — even when you download a new build into a completely separate folder instead of overwriting.

How resolution works (advanced)
  1. $SWEEPYCL_USERDATA_DIR environment variable
  2. $SWEEPYCLAUDE_USERDATA_DIR (legacy env var)
  3. The pointer file at — set by this dialog and survives across version installs
  4. SweepyCL_userdata sibling folder next to the build
  5. SweepyClaude_userdata (legacy sibling folder)
  6. Inside the build folder itself (not recommended — will be lost on upgrade)
Tip — the folder doesn't have to exist yet; it'll be created. A good convention is a folder named SweepyCL_userdata next to your build folder.
